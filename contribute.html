🚀 Kuinka osallistua?

1

Fork Repository

Tee oma kopio projektista GitHub-tilillesi klikkaamalla "Fork"-nappia.

git clone https://github.com/[sinun-käyttäjänimi]/LexIT.git
2

Luo uusi branch

Tee muutoksesi omassa branchissa. Käytä kuvaavaa nimeä:

git checkout -b add-new-terms git checkout -b fix-search-bug git checkout -b improve-ui
3

Tee muutokset

Muokkaa tiedostoja:

  • Termien lisäys: Päivitä js/script.js tiedoston itTerms-array
  • Kategoriat: Lisää termi sopivalle HTML-sivulle
  • Tyylittelyt: Muokkaa css/styles.css tiedostoa
4

Testaa muutokset

Varmista että sivusto toimii:

  • Avaa index.html selaimessa
  • Testaa hakutoiminto uusille termeille
  • Tarkista että navigaatio toimii
  • Varmista responsiivisuus
5

Commit ja Push

Tallenna muutokset:

git add . git commit -m "Add new AI terms and improve search" git push origin add-new-terms
6

Lähetä Pull Request

Mene GitHub-repositoryyn ja klikkaa "Compare & pull request". Kirjoita selkeä kuvaus muutoksistasi.

📝 Osallistumisohjeet

✅ Hyviä käytäntöjä

  • Selkeät commit-viestit: "Add DNS and DHCP terms" on parempi kuin "update"
  • Yksi asia kerrallaan: Keskity yhteen asiaan per Pull Request
  • Testaa ensin: Varmista että kaikki toimii ennen lähettämistä
  • Seuraa tyyliä: Käytä samaa koodistyliä kuin projektissa
  • Suomenkielisyys: Pidä käyttöliittymä suomenkielisenä

📋 Termien lisäysohjeet

  • Aakkosjärjestys: Lisää uudet termit oikeaan aakkosjärjestykseen
  • Oikea kategoria: Valitse sopivin kategoria termille
  • Tarkkuus: Varmista että käännös on oikea
  • Johdonmukaisuus: Käytä samaa muotoilua kuin muissa termeissä
